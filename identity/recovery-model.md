# Identity Recovery Model

## Objective

Ensure account recovery without weakening security posture.

---

## Recovery Methods

Primary:

- Hardware security key

Secondary:

- Backup hardware key

Tertiary:

- Recovery codes

---

## Recovery Code Storage

Stored:

- Offline on paper
- Secure physical location

Not stored:

- Email
- Cloud storage
- Plaintext files

---

## Disaster Scenarios Covered

- Hardware key loss
- Device theft
- Device failure

---

## Security Principle

Recovery mechanisms must not weaken authentication strength.

---

## Recovery Hierarchy

Hardware key → Backup key → Recovery code
